Kap 3.8 – Generiska klasser och metoder

Skriv återanvändbar kod som kan fungera med flera olika datatyper utan att tappa tydlighet.

Mål med lektionen

När du har arbetat klart med denna lektion ska du:

  • Förstå vad generisk kod betyder i Python.
  • Kunna skriva enkla generiska funktioner och metoder med TypeVar.
  • Kunna skapa en enkel generisk klass med Generic.
  • Förstå när generiska klasser och metoder gör kod mer återanvändbar och testbar.

Så här lär du dig bäst

Börja med att jämföra två funktioner som gör samma sak för olika datatyper. Fundera sedan på vad som egentligen är gemensamt. Generiska typer är inte främst till för att göra koden svårare, utan för att beskriva tydligare vilka typer som hänger ihop.

Centrala begrepp

  • Generisk kod: kod som kan användas med flera datatyper.
  • Typvariabel: en symbolisk typ, ofta T, som betyder "samma typ som används här".
  • TypeVar: används för att skapa en typvariabel i Python.
  • Generic: används när en klass ska vara generisk.
  • Typhint: en markering som hjälper läsaren och editorn förstå vilka typer koden arbetar med.

Begreppen förklarade enklare

Generisk kod

Generisk kod betyder att du skriver samma lösning en gång, men låter den fungera med flera typer. Funktionen first() behöver till exempel inte veta om listan innehåller namn, poäng eller bokobjekt. Den gör alltid samma sak: hämtar första värdet.

Typvariabeln T

T är inte en vanlig variabel som får ett värde när programmet körs. Den är en typmarkering. Den betyder: "den typ som används här ska vara samma typ på de andra platserna där T står".

from typing import TypeVar

T = TypeVar("T")

def first(values: list[T]) -> T:
    return values[0]

Om values är list[str] betyder T text, alltså str. Om values är list[int] betyder T heltal, alltså int. Poängen är sambandet: funktionen returnerar samma typ som listan innehåller.

TypeVar

TypeVar("T") skapar typvariabeln T. Utan den vet inte Python eller editorn att T är tänkt att vara en typ.

from typing import TypeVar

T = TypeVar("T")

Generic[T]

Generic[T] används när en hel klass ska arbeta med en typ som bestäms senare. Klassen Register kan till exempel vara ett register för böcker, elever eller recept, men ett specifikt register bör hålla sig till en typ åt gången.

book_register = Register[Book]()
student_register = Register[Student]()

Register[Book] betyder: detta register är tänkt att lagra Book-objekt. Register[Student] betyder: detta register är tänkt att lagra Student-objekt.

Typhintar kör inte programmet åt dig

Typhintar som list[T], -> T och Register[Book] är främst hjälp för människor, editorer och typkontrollverktyg. De gör koden tydligare, men Python stoppar normalt inte automatiskt alla typfel när programmet körs.

Varför behövs generiska typer?

Tänk att du vill skriva en funktion som hämtar första värdet ur en lista. Funktionen fungerar på samma sätt oavsett om listan innehåller tal, text eller objekt.

def first_text(values: list[str]) -> str:
    return values[0]

def first_number(values: list[int]) -> int:
    return values[0]

Det här ger duplicerad kod. Logiken är identisk. Det enda som skiljer är typen. Med en generisk funktion kan vi beskriva samma idé en gång.

Generisk funktion

from typing import TypeVar

T = TypeVar("T")

def first(values: list[T]) -> T:
    return values[0]

name = first(["Adam", "Bo", "Cia"])
score = first([10, 20, 30])

print(name)
print(score)

T betyder att funktionen ska returnera samma typ som listan innehåller. Om listan är list[str] blir returvärdet str. Om listan är list[int] blir returvärdet int.

Generisk funktion med flera parametrar

Samma idé kan användas i metoder, men exemplet nedan är en fristående funktion för att göra grunden tydlig. Funktionen kontrollerar om ett värde finns i en lista, oavsett vilken typ listan innehåller.

from typing import TypeVar

T = TypeVar("T")

def contains_value(value: T, values: list[T]) -> bool:
    return value in values

print(contains_value("Adam", ["Adam", "Bo"]))
print(contains_value(3, [1, 2, 3]))

Det viktiga är kopplingen mellan value och values: de ska höra ihop typmässigt. När value är en sträng ska listan också vara en lista med strängar. När value är ett heltal ska listan vara en lista med heltal. Det gör koden lättare att förstå och bättre för editorstöd.

Generisk klass

En generisk klass passar när klassen ska kunna lagra eller hantera olika typer av objekt, men med samma logik. Här är en enkel låda som kan innehålla vad som helst.

from typing import Generic, TypeVar

T = TypeVar("T")

class Box(Generic[T]):
    def __init__(self, content: T):
        self._content = content

    def get(self) -> T:
        return self._content

text_box = Box("Hej")
number_box = Box(42)

print(text_box.get())
print(number_box.get())

Box[str] betyder en låda som innehåller text. Box[int] betyder en låda som innehåller heltal. Python tvingar normalt inte detta vid körning, men typhintarna hjälper programmeraren, editorn och testningen.

Exempel: generiskt register

I tidigare kapitel har du arbetat med böcker, elever och recept. Ett register kan ofta ha samma grundlogik oavsett vilken typ av objekt som lagras.

from typing import Generic, TypeVar

T = TypeVar("T")

class Register(Generic[T]):
    def __init__(self):
        self._items: list[T] = []

    def add(self, item: T) -> None:
        self._items.append(item)

    def all_items(self) -> list[T]:
        return self._items

class Book:
    def __init__(self, title: str):
        self.title = title

book_register = Register[Book]()
book_register.add(Book("Python"))

first_book = book_register.all_items()[0]
print(first_book.title)

Samma Register-klass kan användas för Book, Student, Recipe eller andra objekt. Det minskar duplicering och gör klassens ansvar tydligt: den hanterar en samling objekt.

När ska du använda generiska klasser och metoder?

  • När samma logik ska fungera för flera datatyper.
  • När du vill undvika duplicerade funktioner som bara skiljer sig i typ.
  • När du vill att editorn ska kunna ge bättre stöd och varna för fel typ.
  • När du bygger egna samlingar, register, köer eller lagringsklasser.

När behövs det inte?

  • När koden bara används på ett enda ställe.
  • När en vanlig funktion eller klass är tydligare.
  • När generiska typer gör lösningen svårare att läsa än problemet kräver.

Sammanfattning

  • Generisk kod kan återanvändas med flera datatyper.
  • TypeVar beskriver en typ som återkommer på flera platser.
  • Generic[T] används när en klass ska fungera med valfri men sammanhängande typ.
  • Generiska klasser och metoder kan göra kod mer läsbar, testbar och återanvändbar.
